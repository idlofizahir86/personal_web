API Documentation

by Idlofi Zahir Rajaba

ENDPOINT

https://akunkeun.lldikti4.id

Method: GET
Description: Mengambil data RPD (Rencana Perjalanan Dinas) berdasarkan ID.
Parameters: id (path): ID RPD yang ingin diambil.
Authentication: Authenticated user with role pegawai.
Response: 
{
  "id": 123,
  "nama": "Perjalanan Dinas Jakarta",
  "tanggal": "2024-08-23"
}

Method: GET
Description: Mengambil daftar semua data perjadin.
Authentication: Authenticated user with role pegawai.
Response: 
[
  {
    "id": 123,
    "nama": "Perjalanan Dinas Jakarta",
    "tanggal": "2024-08-23"
  },
  {
    "id": 124,
    "nama": "Perjalanan Dinas Bandung",
    "tanggal": "2024-08-24"
  }
]

Method: GET
Description: Mengambil dokumen berdasarkan nama file.
Parameters: filename (path): Nama file dokumen yang ingin diambil.
Authentication: Authenticated user with role pegawai.
Response: Dokumen dalam format yang sesuai (misalnya PDF, gambar).

Method: GET
Description: Mengambil detail perjadin berdasarkan ID.
Parameters: id (path): ID perjadin yang ingin diambil.
Authentication: Authenticated user with role pegawai.
Response: Detail perjadin dalam format JSON.

Method: GET
Description: Mengambil data untuk langkah kedua perjadin berdasarkan ID.
Parameters: id (path): ID perjadin yang ingin diambil.
Authentication: Authenticated user with role pegawai.
Response: Data langkah kedua perjadin dalam format JSON.

Method: GET
Description: Mengambil riwayat perjadin berdasarkan status.
Parameters: status (path): Status perjadin (misalnya "selesai", "proses").
Authentication: Authenticated user with role pegawai.
Response: Daftar riwayat perjadin dalam format JSON.

Method: GET
Description: Mengambil detail perjadin tertentu berdasarkan ID.
Parameters: id (path): ID perjadin yang ingin diambil.
Authentication: Authenticated user with role pegawai.
Response: Detail perjadin dalam format JSON.

Method: GET
Description: Mengambil catatan perjadin berdasarkan ID.
Parameters: id (path): ID perjadin yang ingin diambil.
Authentication: Authenticated user with role pegawai.
Response: Catatan perjadin dalam format JSON.

Method: GET
Description: Mengambil catatan perjadin untuk pengguna berdasarkan ID.
Parameters: id (path): ID perjadin yang ingin diambil.
Authentication: Authenticated user with role pegawai.
Response: Catatan perjadin untuk pengguna dalam format JSON.

Method: POST
Description: Menyimpan data perjadin baru.
Authentication: Authenticated user with role pegawai.
Request Body: 
{
  "nama": "Perjalanan Dinas Jakarta",
  "tanggal": "2024-08-23"
}
Response: 
{
  "success": true,
  "message": "Perjadin berhasil disimpan.",
  "id": 123
}

Method: PUT
Description: Memperbarui status peserta berdasarkan ID.
Parameters: id (path): ID peserta yang ingin diperbarui.
Authentication: Authenticated user with role pegawai.
Request Body: 
{
  "status": "aktif"
}
Response: 
{
  "success": true,
  "message": "Status peserta berhasil diperbarui."
}

Method: PUT
Description: Memperbarui detail perjadin.
Authentication: Authenticated user with role pegawai.
Request Body: Data detail perjadin baru dalam format JSON.
Contoh Request Body: 
{
  "id": 123,
  "judul": "Perubahan Detail Perjadin",
  "tanggal": "2024-08-23",
  "lokasi": "Jakarta",
  "deskripsi": "Deskripsi baru perjadin."
}

Response: Status pembaruan.
Contoh Response: 
{
  "success": true,
  "message": "Detail perjadin berhasil diperbarui."
}

Method: DELETE
Description: Menghapus data peserta berdasarkan ID.
Parameters: HPeserta (path): ID peserta yang ingin dihapus.
Authentication: Authenticated user with role pegawai.
Response: 
{
  "success": true,
  "message": "Peserta berhasil dihapus."
}

Method: DELETE
Description: Menghapus data fasilitas perjadin berdasarkan ID.
Parameters:
  • HFasilitasPerjadin (path): ID fasilitas perjadin yang ingin dihapus.
Authentication: Authenticated user with role pegawai.
Response: Status penghapusan.
Contoh Response:

  "message": "Fasilitas perjadin berhasil dihapus."
}
